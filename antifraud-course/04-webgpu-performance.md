## Краткое содержание

Представлена новейшая технология WebGPU Atomic Fingerprint, анонсированная на конференции V-Sec (США, июнь 2025). Это революционный метод идентификации пользователей через GPU-планировщик, который придёт на замену устаревшему WebGL fingerprint.

---

## Ключевые тезисы

### 1. BrowserLeaks WebGPU Hash - бесполезен

Hash на BrowserLeaks НЕ является реальным fingerprint:
- Базируется на константах обработки графики
- Константы одинаковы для большинства видеокарт
- Невозможно отличить поколения видеокарт
- Модель видеокарты не раскрывается

### 2. WebGPU Atomic Fingerprint

| Параметр | Значение |
|----------|----------|
| Точность идентификации | 70% |
| Ресурсоёмкость | Минимальная |
| Обнаружение | Очень сложно |
| Размер кода | ~553 строки |
| Ожидаемое внедрение | Октябрь-ноябрь 2025 |

### 3. Как работает

Технология использует **GPU-планировщик** (GPU Scheduler):
- Запуск Compute Shaders
- Параллельная операция Atomic Add
- Анализ распределения потоков
- Уникальность зависит от видеокарты + драйверов

## Полный конспект

### Проблема текущих WebGPU-чекеров

BrowserLeaks показывает hash, который:
- Формируется из констант обработки графики
- Одинаков для разных поколений видеокарт
- Не отличает GTX 1080 от RTX 4090
- Не является реальным fingerprint

### WebGPU Atomic Fingerprint (V-Sec 2025)

#### Технология

1. Запускаются **Compute Shaders WebGPU**
2. Множество потоков выполняют **Atomic Add** параллельно
3. **GPU-планировщик** распределяет ресурсы между потоками
4. На разных видеокартах планировщик работает по-разному
5. Драйверы также влияют на распределение

#### Что определяет

- Модель видеокарты
- Версию драйверов (старые vs новые)
- Встроенная vs дискретная GPU
- Ноутбук vs десктоп
- Использование VPS (нет GPU)

#### Точность

**70%** - отличный показатель, потому что:
- Комбинируется с User-Agent
- Комбинируется с тестом производительности
- В сумме = высокая идентификация

### Незаметность теста

При тестировании на реальной системе:
- Нагрузка GPU: 40-50% (без изменений)
- При запуске теста: **никаких видимых изменений**
- Обнаружить тест крайне сложно

### Методы обхода

#### Неправильный способ (антидетекты)

Модификация исходного кода браузера:
- Изменение количества потоков (10 -> 12 или 8)
- Рандомизация ресурсами браузера

**Проблема:** Видно, сколько потоков задействовано. Детект.

#### Правильные способы

1. **Изменение режима питания**
   - GPU-планировщик чувствителен к power settings
   - Разные режимы = разные fingerprint

2. **Настройки видеокарты**
   - NVIDIA Control Panel -> Per-Application Settings
   - Изменить распределение ресурсов для браузера
   - Максимальная производительность vs Визуальное качество

3. **Модификация драйверов** (сложно)
   - Применимо для виртуальных машин
   - Требует переподписи драйверов
   - Риск повреждения системы

### Обнаружение на сайтах

#### Если код не обфусцирован

```javascript
// Искать в исходниках:
navigator.gpu
```

#### Если код обфусцирован

Отключить WebGPU:
```
chrome://flags -> WebGPU -> Disabled
```

Если сайт работает по-другому = использует WebGPU.

### Практические последствия

Если вы:
- Заявляете RTX 2080 в User-Agent
- Работаете на VPS без видеокарты
- Используете антидетект без реальной GPU

**Результат:** Atomic Fingerprint покажет несоответствие = проблемы с антифродом.

## Технические детали

### Compute Shaders

```
GPU -> Compute Shaders -> Atomic Add (parallel threads)
         |
         v
    GPU Scheduler
         |
         v
    Thread Distribution
         |
         v
    Unique Fingerprint
```

### Факторы влияния

| Фактор | Влияние на fingerprint |
|--------|------------------------|
| Модель GPU | Высокое |
| Версия драйверов | Среднее |
| Power Mode | Среднее |
| App Settings | Среднее |
| Температура GPU | Низкое |
