# 08. AudioContext Fingerprint

**Автор:** Вектор Т13
**Длительность:** ~20 минут
**Уровень:** Продвинутый
**Тип урока:** Разоблачение мифов + Performance Benchmark

---

## Краткое содержание

AudioContext fingerprint на 99% НЕ имеет отношения к аудио. Это отпечаток **процессора**, а не звуковой карты. Менять его бесполезно и даже вредно. Реальная ценность - в **бенчмарке производительности**, который определяет VPS, VM и ботов.

---

## Ключевые тезисы

### 1. Мифы об AudioContext

| Миф | Реальность |
|-----|------------|
| Это отпечаток звуковой карты | Работает без звуковой карты |
| Нужно менять хэш | Хэш одинаков на Chrome 98-138 |
| Зависит от колонок | Нет зависимости |
| Уникален для каждого ПК | Одинаков на большинстве CPU |

### 2. Audio Benchmark

| Среда | Время выполнения |
|-------|------------------|
| Реальный ПК | 1.5-2.5 мс |
| VPS (шаринг ядер) | 45-50 мс |
| Виртуальная машина | 45-50 мс |
| Сервер с низкой частотой | 20-35 мс |

### 3. Главный вывод

**Подменять AudioContext хэш = идиотизм. Важна только производительность.**

## Полный конспект

### История AudioContext

- **2013** - создан
- **2015** - обнаружен студентами Принстона на Google/Facebook
- **2018** - сайт audiofingerprint.com закрылся
- **Сейчас** - всё ещё используется для детекта ботов

### AudioContext НЕ использует аудио

> "Если вы возьмёте компьютер без аудиокарты и выполните тест - он будет успешен."

Почему:
- Не воспроизводит звук
- Не записывает звук
- Выполняется **внутри браузера**
- Это отпечаток **аудиодвижка браузера**

### Стабильность хэша

Один и тот же хэш на:
- Chrome 98-138 (40+ версий!)
- Opera
- Edge
- Все Chromium-браузеры

**Вывод:** Если движок не меняется - хэш не меняется.

### Почему чекеры врут

Владельцы чекеров используют AudioContext в маркетинге:
- Легко подменить (даже через реестр)
- Пользователи не понимают технологию
- Платят за "уникальный" отпечаток

**Реальность:** Менять бесполезно - нет вариативности.

### Реальный отпечаток: CPU

AudioContext зависит от:
- Скорости выполнения инструкций CPU
- Тактовой частоты процессора
- Типа процессора (десктопный vs серверный)

#### Вариативность

| Частота дискретизации | Популярность |
|----------------------|--------------|
| 48000 Hz | Стандарт |
| 44100 Hz | Часто |
| Другие | 0.0001% |

**Максимум:** 2 популярных + 6 редких отпечатков на один ПК.

### Audio Benchmark - реальный детект

#### Как работает

```
WebGL: FPS падает на VM/VPS
Audio: Время выполнения РАСТЁТ на VM/VPS
```

#### Результаты

| Среда | Время | Статус |
|-------|-------|--------|
| Реальный ПК (i7) | 2.30 мс | OK |
| VPS (shared) | 45-50 мс | Детект |
| VirtualBox | 45-50 мс | Детект |
| VMware | 45-50 мс | Детект |
| Старый сервер (2 GHz) | 20-35 мс | Детект |
| Сервер с высокой частотой | 2-5 мс | OK |

### Что определяет Audio Benchmark

#### Определяет (плохо)

1. **VPS** - шаринг ядер = медленно
2. **Виртуальные машины** - эмуляция = медленно
3. **Антидетект на VPS** - двойная эмуляция
4. **Серверы с низкой частотой** - Intel Xeon старый
5. **VectorT13 Antidetect до v5**

#### НЕ определяет (хорошо)

1. **EFI Spoofers**
2. **Серверы с высокой частотой**
3. **Антидетект на реальном ПК**
4. **Антидетект с CPU Boost**

### Серверы - важен CPU

> "Пользователи обращают внимание на количество ядер, но не на тактовую частоту."

| Сервер | Результат |
|--------|-----------|
| Intel Xeon 2 GHz (старый) | Детект |
| Intel Xeon 3.5+ GHz | OK |
| AMD EPYC высокочастотный | OK |

### CPU Boost для виртуальных машин

Два способа:
1. **Трассировка** - инструкции на реальный CPU
2. **Тайминги** - модификация исходников VM

**Доступно:** Только в VectorT13 Antidetect v5+

### Как обнаружить AudioContext на сайте

Плагины для Firefox/Chrome:
- Определяют использование AudioContext API
- **Но:** нельзя увидеть через Performance tab (16-20 мс = норма)

### Важное замечание

> "Когда видите AudioContext на сайте - не думайте, что это fingerprint. В 99% случаев это антибот-проверка производительности."

## Технические детали

### Параметры AudioContext

| Параметр | Влияние на хэш | Влияние на benchmark |
|----------|----------------|---------------------|
| Sample Rate | Да | Нет |
| Channel Count | Нет | Нет |
| Bitrate | Нет | Нет |

### Sample Rate варианты

```
48000 Hz - стандарт Windows
44100 Hz - альтернатива
192000 Hz - редко
96000 Hz - редко
...и ещё 4 редких
```

### Формула детекта

```
if (audioTime > 40ms) {
    return "VM/VPS/Bot";
} else if (audioTime > 15ms) {
    return "Server/Old CPU";
} else {
    return "Real PC";
}
```

## Ключевые выводы

1. **AudioContext != аудио** - работает без звуковой карты
2. **Хэш одинаков** - Chrome 98-138, Edge, Opera
3. **Подмена бесполезна** - нет вариативности
4. **Benchmark = детект** - время выполнения выдаёт VM/VPS
5. **Частота CPU критична** - низкая = детект сервера
6. **VPS на антидетекте = глупо** - двойная эмуляция
7. **Чекеры врут** - маркетинг вокруг бесполезной подмены
