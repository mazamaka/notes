## 10 методов определения геолокации

### 1. IP-based геолокация (первобытный метод)
- Проверка IP через базы Maxmind
- ASN сети — конгломерат IP-адресов (~350 000 ASN в мире)
- ASN важнее IP для определения хостингов vs резидентных адресов
- Единственный метод до 2007 года
- **Обход:** Прокси, VPN

### 2. Trace Route / TTL анализ
- TTL (Time to Live) = 128 стандартно для Windows
- Уменьшается на 1 на каждом узле
- Веб-ресурсы могут определить расстояние по TTL
- TRCROAD работает только локально

### 3. DNS-based геолокация
- Определение региона по используемому DNS-серверу
- Утечки DNS при неправильной настройке VPN

### 4. Timezone / часовой пояс
- Сравнение timezone браузера с IP-геолокацией
- Несоответствие = признак подмены

### 5. Языковые настройки
- Accept-Language headers в HTTP
- Системный язык vs IP-регион

### 6. WebRTC утечки
- Раскрытие реального IP через STUN-серверы
- Обход NAT выдаёт локальные адреса

### 7. JavaScript API геолокации
- navigator.geolocation API
- Требует разрешения пользователя
- GPS/WiFi/Cell tower данные

### 8. Canvas/WebGL fingerprinting
- Аппаратные отпечатки GPU
- Косвенное определение региона по железу

### 9. IPv6 утечки
- 98% VPN работают только с IPv4
- IPv6 трафик идёт напрямую

### 10. WiFi BSSID геолокация
- База MAC-адресов точек доступа
- Google Street View собирал эти данные

## Ключевые выводы

1. **Maxmind** — основа всей IP-геолокации, бесплатная проверка на их сайте
2. **ASN** важнее IP — позволяет определить тип подключения (хостинг/резидент)
3. **98% VPN** не защищают от IPv6 утечек
4. **Публичные VPN** ненадёжны — логи ведут все
5. **Оффшор не защита** — серверы можно изъять/взломать

## Рекомендации
- Поднимать VPN самостоятельно
- Использовать резидентные прокси вместо датацентровых
- Проверять геолокацию на Maxmind, а не на чекерах
- Настраивать kill-switch для предотвращения утечек
