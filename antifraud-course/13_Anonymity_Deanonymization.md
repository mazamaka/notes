# Ультимативная анонимность: Как вас деанонимизируют

**Автор:** Vektor T13
**Источник:** YouTube вебинар
**Тема:** Комплексная защита анонимности в 2026 году
**Формат:** 92 слайда, 4 основных блока

## Структура вебинара

1. **Браузерная анонимность**
2. **Аппаратное обеспечение** (мобильные устройства, Apple)
3. **Сетевая часть** (наибольшее количество ошибок пользователей)
4. **Криптовалюты** (анонимность и риски)

## Ключевые концепции

### Парадокс анонимности
- Безопасность БЕЗ анонимности может существовать
- Анонимность БЕЗ безопасности — невозможна
- Анонимность обратно пропорциональна удобству

### Принцип балаклавы
- Излишняя анонимность → уникализация
- Нужно выглядеть как "реальный человек"
- Цель: качественная маска, а не балаклава

### Выбор инструментария
- Против кого защищаемся? (конкуренты, налоговая, спецслужбы)
- Больше "презервативов" ≠ лучше защита
- Правильная связка = 15 минут на создание

## Браузерная анонимность

### Корпоративные браузеры = риск
- Chrome, Edge, Opera, Firefox — все одинаково опасны
- **Brand-code** — уникальный идентификатор при скачивании
- **XCH data** — трекер в HTTP-протоколе
- **Встроенный DNS** — резервный путь через сервера создателей

### Телеметрия
- Встроенная функция, НЕ опция
- Резервный путь при блокировке прямого коннекта
- Нельзя полностью отключить

### Что собирают браузеры
- Телеметрия использования
- Сетевое окружение (MDNS сканирование локальной сети)
- Список установленного ПО
- Информация о железе
- Данные ОС (редакция, языки, обновления)
- Интересы, привычки, поведение

### Рекомендации по браузерам
- **Ungoogled Chrome** — единственный рекомендуемый (открытый код на GitHub)
- **Brave** — НЕ рекомендуется (маркетинг ≠ реальная защита)
- Firefox — больше не "рыцарь в сияющих доспехах"
- Плагины на Firefox — "всё равно [ __ ] говнище"

### Система загрузки в Antidetect
- Ранее: cURL (с сентября 2025 считается признаком бота)
- Сейчас: Windows BITS (менее стабильно, но без санкций Google)

## Сетевая часть

### VPN проблемы
- 98% VPN не обрабатывают IPv6
- DNS утечки при неправильной настройке
- Все VPN ведут логи (нет смысла не вести)
- Невозможно проверить, ведёт ли VPN логи

### Оффшор — не защита
- NordVPN зарегистрирован в Панаме
- Но серверы в других юрисдикциях
- Данные можно получить через партнёров

## Инструменты деанонимизации

### Ультразвуковые маяки
- Невидимые для человека звуки в видео/аудио
- Микрофоны устройств улавливают
- Используется для поимки педофилов
- TPM чипы, Apple Secure Enclave

### Cross-device tracking
- Связывание устройств одного пользователя
- Chromecast/MDNS сканирование сети
- MAC-адреса устройств в локальной сети

## Выводы

1. **Анонимность 2026** принципиально отличается от 2025
2. **Шапочка из фольги** — не паранойя, а необходимость
3. **Простые решения** (Tails/Whonix/Qubes) требуют переоценки
4. **Цена анонимности** — потеря удобства
5. **Правильная связка** создаётся за 15 минут

## Практические рекомендации

- Не использовать корпоративные браузеры для анонимности
- Проверять Brand-code при скачивании Chrome
- Использовать Ungoogled Chrome или специализированные решения
- Настроить kill-switch для VPN
- Блокировать IPv6 если VPN не поддерживает
- Менять Brand-code в Application Data
