## О проекте

Виртуальный антидетект-роутер — альтернатива программным проксификаторам с более правильным формированием сетевого трафика.

**Проблема обычных проксификаторов:**
- Не только смена IP — важны частота запросов, структура трафика
- Пользователи не могут обойти даже антибот-систему
- Устройства воспринимаются как боты

**Преимущества роутера:**
- Полная поддержка IPv6, DNS, WebRTC, WebTransport, QUIC
- MTU 1500 (как у реального пользователя)
- Мимикрирование под реальное соединение

## Технические требования

### Компоненты
1. **Скрипт установки** — доступен на GitHub
2. **OpenWRT 24.10.4** — прошивка для роутеров
3. **VPS сервер** — рекомендуется в целевой стране

### Выбор VPS
- **Локация:** В той стране, по которой работаете (снижение Latency/Jitter)
- **Скорость:** Premium Intel ($8/мес) = 1 Гбит vs базовый ($4/мес) = 100 Мбит
- **Образ:** x86-64, Generic EXT4 Combined EFI

### Пример на Digital Ocean
- Digital Ocean не поддерживает IPv6 для кастомных образов
- SSH ключ — формальность (OpenWRT не использует)
- Recovery Console для первоначальной настройки

## Пошаговая установка

### 1. Создание дроплета
- Выбрать датацентр
- Custom Images → загруженный OpenWRT образ
- Premium Intel для 1 Гбит канала

### 2. Первоначальная настройка (через Recovery Console)

### 3. SSH подключение
```bash

```

### 4. Установка скрипта
- Скрипт с GitHub
- Автоматическая настройка фаервола и маршрутизации

### 5. Настройка VPN
- Веб-интерфейс OpenWRT
- Вкладка VPN → OpenVPN
- Загрузка конфигурации (.ovpn)
- Для auth-user-pass: указать путь к файлу с логином/паролем

## Архитектура VPN

### Два типа подключений
1. **Клиент → Сервер (RW):** Без шифрования (экономия CPU)
2. **Сервер → Прокси:** Шифрование контролируется принимающей стороной

**Почему без шифрования:**
- Шифрование нагружает процессор
- Снижает скорость соединения
- Для большинства задач безопасность избыточна
